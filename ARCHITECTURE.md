# Project Architecture: Atlas 

## Overview
This project aims to map the entirety of modern mathematics into a rigorous, interactive Directed Acyclic Graph (DAG). It serves two functions:
1. **An Encyclopedia:** A visual, explorable map of theorems, lemmas, and proofs, pairing rigorous formal structure with pedagogical intuition.
2. **A Proof-Writing IDE:** A local, web-based environment that uses deterministic link-prediction algorithms to help mathematicians discover logical connections as they write.

---

## 1. The Source of Truth (Data Layer)
All mathematical content is written in **Typst** and stored as standard text files in a Git repository. 

Contributors do not write standard documents; they use a strict set of custom Typst functions that act as our database schema. 

### The Schema
```typst
#import "@local/math-graph:0.1.0": theorem, intuition, proof

#theorem(
  id: "thm-bolzano-weierstrass",
  name: "Bolzano-Weierstrass Theorem",
  tags: ("analysis", "topology"),
  deps: ("def-bounded-sequence", "thm-monotone-subsequence"),
  status: "expert-verified"
)[
  Every bounded sequence in $RR^n$ has a convergent subsequence.
]

#intuition(for: "thm-bolzano-weierstrass")[
  Imagine a completely full box of particles. If you keep dividing the box into smaller halves, at least one of those halves must contain infinitely many particles...
]

#proof(for: "thm-bolzano-weierstrass")[
  Let $(x_n)$ be a bounded sequence in $RR^n$...
]
```
*Note: The metadata arguments (`id`, `deps`, `status`) represent the strict database schema, while the content brackets `[]` allow for completely Typst markup.*

## 2. The Build Engine (Rust Parser)
We are not using a traditional database. Instead, we use a custom Rust CLI tool that leverages the official `typst` crate. 

During the CI/CD build step (GitHub Actions), this Rust parser will:
1. Parse every `.typ` file in the repository to generate an Abstract Syntax Tree (AST).
2. Traverse the AST to locate all schema functions (`#theorem`, `#lemma`, etc.).
3. Extract the metadata (`id`, `deps`) and serialize it into a global `graph.json` file.
4. Instruct the Typst compiler to render the content blocks into standalone **static HTML fragments** (e.g., `thm-bolzano-weierstrass.html`).
5. Generate a `search_index.json` using a TF-IDF/BM25 algorithm for local text search.

### Expected `graph.json` Output Schema
```json
{
  "nodes": [
    {
      "id": "thm-bolzano-weierstrass",
      "label": "Bolzano-Weierstrass Theorem",
      "status": "expert-verified"
    }
  ],
  "edges": [
    { "source": "thm-bolzano-weierstrass", "target": "def-bounded-sequence" },
    { "source": "thm-bolzano-weierstrass", "target": "thm-monotone-subsequence" }
  ]
}
```

## 3. The Frontend (WebGL + HTMX)
The frontend is a completely static website (hosted on Cloudflare Pages/GitHub Pages) consisting of a split-screen UI.

### The Left Panel: The Graph (WebGL)
* Fetches the static `graph.json` on load.
* Uses a WebGL rendering library (e.g., Sigma.js or VivaGraph) to draw the DAG.
* **Interaction:** Clicking a node triggers an HTMX event to update the Right Panel.

### The Right Panel: The Content (HTMX)
* An empty `<div id="math-content"></div>`.
* Uses **HTMX** to fetch the pre-compiled static HTML fragments generated by the Rust parser.
* Provides instantaneous reading without heavy JavaScript state management.
* E.g., `<button hx-get="/compiled/thm-bolzano-weierstrass.html" hx-target="#math-content">`

---

## 4. The Local IDE & Suggestion Engine (Client-Side)
To provide a "Copilot for Proofs" without server costs or AI hallucinations, the proof-writing environment runs locally in the browser (via WebAssembly and Vanilla JS).

When a user writes a draft:
1. **Instant Render:** The Typst WASM compiler renders the math live.
2. **Deterministic Link Prediction:** A JavaScript worker runs classic Graph Theory algorithms (like Adamic-Adar or Jaccard Coefficient) against the locally downloaded `graph.json`. If a user cites `@lemma-A` and `@lemma-B`, the engine suggests nodes that frequently co-occur with A and B.
3. **Local Search:** Queries are run against the downloaded `search_index.json` using standard BM25 ranking.

*Strict Requirement: No Large Language Models (LLMs) or non-deterministic generative tools are to be used in this pipeline.*

## 5. Development Roadmap & Division of Labor

* **Task 1 (The Parser):** Build the Rust CLI tool that traverses a Typst AST and outputs `graph.json` and HTML fragments. *(Ideal for CS Undergrads)*
* **Task 2 (The Renderer):** Build the static `index.html` file integrating WebGL for the canvas and HTMX for the side panel. *(Ideal for CS Undergrads)*
* **Task 3 (The Infrastructure):** Set up GitHub Actions for the build step, Cloudflare Pages for hosting, and repository branch protection rules. *(Ideal for Senior Devs)*
* **Task 4 (The Suggestion Engine):** Implement the WASM Typst editor and the local JavaScript Adamic-Adar link prediction algorithm. *(Ideal for CS Undergrads/PhDs)*
