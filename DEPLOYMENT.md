# Atlas Deployment & Workflow Guide

This guide covers the complete setup for running Atlas with the hybrid CLI/web server architecture.

## Architecture Overview

Atlas now supports two modes of operation:

1. **Compilation Mode** (default): Parses all `.typ` files, extracts math nodes, and generates static HTML fragments
2. **Server Mode**: Runs as a web server to accept submissions from contributors

```
Atlas/
├── compiler/          # Rust backend (handles both modes)
├── math/
│   ├── submissions/   # User submissions land here
│   ├── schema/        # Core Typst schema
│   └── algebra/       # Your math content
└── public/
    ├── index.html     # Main graph viewer
    ├── submit.html    # Contributor portal
    ├── json/
    │   └── graph.json # Generated by compiler
    └── nodes/         # Generated SVG/PDF fragments
```

## Local Development Setup

### Prerequisites

- **Rust** (1.70+): [https://rustup.rs/](https://rustup.rs/)
- **Typst CLI**: `cargo install typst-cli`
- **Git**: For version control and commits

### Step 1: Build the Compiler

```bash
cd /home/mithril/code/Atlas/compiler
cargo build --release
```

This creates the binary at `compiler/target/release/math-graph-compiler`.

### Step 2: Run Initial Compilation

```bash
cd compiler
cargo run --release
```

This will:
1. Parse all `.typ` files in `../math/`
2. Generate `../public/json/graph.json`
3. Compile each node to SVG and PDF in `../public/nodes/`

### Step 3: Start the Web Server

In a new terminal (or use `screen`/`tmux`):

```bash
cd compiler
cargo run --release -- server
```

The server will bind to `http://127.0.0.1:3000` and provide these endpoints:

- **GET** `/api/graph` - Fetch the current graph JSON
- **POST** `/api/submit` - Upload a new submission

### Step 4: Open the Web Interface

```bash
# Open the main graph viewer
open http://127.0.0.1:3000/../public/index.html

# Or open the submission portal
open http://127.0.0.1:3000/../public/submit.html
```

## Submission Workflow

### For Contributors

1. Navigate to `http://your-server.com/submit.html`
2. Write your Typst content with the required metadata:
   ```
   // id: thm-unique-id
   // type: theorem
   // deps: [dep1, dep2]
   ---
   
   Your mathematical content in Typst...
   ```
3. Drag & drop the `.typ` file onto the upload box
4. Click "Push to Graph"
5. The server validates, saves, and commits the file

### What Happens Behind the Scenes

1. **Validation**: The submission is tested with `typst compile` before saving
2. **Storage**: File is saved to `math/submissions/{id}.typ`
3. **Version Control**: Git commit is automatically created
4. **Watcher** (optional): A separate process can detect the new file and trigger recompilation

## Production Deployment

### Option 1: VPS with Systemd

Create a systemd service to keep the server running:

**File: `/etc/systemd/system/atlas-server.service`**

```ini
[Unit]
Description=Atlas Math Graph Server
After=network.target

[Service]
Type=simple
User=atlas
WorkingDirectory=/home/atlas/code/Atlas/compiler
ExecStart=/home/atlas/code/Atlas/compiler/target/release/math-graph-compiler server
Restart=on-failure
RestartSec=10

# Security
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

Enable and start:

```bash
sudo systemctl daemon-reload
sudo systemctl enable atlas-server
sudo systemctl start atlas-server

# Monitor logs
sudo journalctl -u atlas-server -f
```

### Option 2: Docker Container

**Dockerfile**

```dockerfile
FROM rust:1.75

RUN apt-get update && apt-get install -y \
    typst \
    git \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app

COPY . .

RUN cd compiler && cargo build --release

EXPOSE 3000

CMD ["compiler/target/release/math-graph-compiler", "server"]
```

Build and run:

```bash
docker build -t atlas-server .
docker run -p 3000:3000 -v /var/atlas/math:/app/math atlas-server
```

### Option 3: Nginx Reverse Proxy

If running on `localhost:3000`, set up Nginx to proxy HTTP requests:

**File: `/etc/nginx/sites-available/atlas`**

```nginx
server {
    listen 80;
    server_name atlas.yourdomain.com;

    # Redirect to HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name atlas.yourdomain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    # Static files (fast)
    location /public {
        root /home/atlas/code/Atlas;
        expires 1d;
        add_header Cache-Control "public, immutable";
    }

    # API endpoints (reverse proxy to Rust)
    location /api {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Submission page
    location /submit.html {
        root /home/atlas/code/Atlas/public;
    }

    # Redirect to index
    location / {
        root /home/atlas/code/Atlas/public;
        try_files $uri /index.html;
    }
}
```

Enable the site:

```bash
sudo ln -s /etc/nginx/sites-available/atlas /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## Git Commit Configuration

For automatic commits to work, ensure your Rust process can authenticate with Git:

### Using SSH Keys

```bash
# Generate if needed
ssh-keygen -t ed25519 -f ~/.ssh/id_atlas

# Add to GitHub/GitLab
cat ~/.ssh/id_atlas.pub

# Configure Git
git config user.email "atlas-bot@yourdomain.com"
git config user.name "Atlas Bot"
```

### Using GitHub Deploy Keys

1. Go to your repository → Settings → Deploy Keys
2. Add the public key from the server
3. Enable "Allow write access"

## Automatic Recompilation (Watcher)

To automatically recompile when new submissions arrive, set up a file watcher:

**Script: `compiler/watch.sh`**

```bash
#!/bin/bash

# Watch the submissions directory for new files
inotifywait -m -r -e create,modify ../math/submissions/ |
while read path action file; do
    echo "Detected change in submissions: $file"
    echo "Recompiling graph..."
    cargo run --release
    echo "Recompilation complete"
done
```

Run in a separate terminal or as a background process:

```bash
chmod +x compiler/watch.sh
./compiler/watch.sh &
```

## Troubleshooting

### Issue: "Permission denied" when writing files

**Solution**: Ensure the Rust process user has write permissions to `math/submissions/`:

```bash
chmod -R 755 math/
chmod -R 777 math/submissions/
```

### Issue: Git commits fail silently

**Solution**: Check Git configuration and SSH keys:

```bash
# Test Git SSH
ssh -T git@github.com

# Configure Git
git config --global user.email "bot@atlas.local"
git config --global user.name "Atlas Bot"
```

### Issue: Typst validation fails for valid submissions

**Solution**: Ensure Typst CLI is installed and the schema path is correct:

```bash
typst --version
ls math/schema/math-graph.typ
```

## Security Considerations

1. **Input Validation**: The submission handler validates Typst syntax before saving
2. **File Permissions**: Store submissions in a directory with restricted access
3. **Rate Limiting**: Consider adding rate limiting to `/api/submit` for production
4. **HTTPS**: Always use HTTPS in production (Nginx + Let's Encrypt)
5. **CORS**: The current setup uses permissive CORS for development; restrict in production

Example CORS restriction in `compiler/src/main.rs`:

```rust
use tower_http::cors::CorsLayer;
use tower_http::cors::Any;

let cors = CorsLayer::permissive()
    // Change to:
    .allow_origin("https://atlas.yourdomain.com".parse().unwrap());

let app = Router::new()
    .route("/api/submit", post(upload_handler))
    .route("/api/graph", get(graph_handler))
    .layer(cors);
```
